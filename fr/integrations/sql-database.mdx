L'intégration de base de données SQL (compatible PostgreSQL) vous permet de connecter une base de données à votre espace de discussion. Une fois connecté, Attlas pourra lire le schéma des tables autorisées et formuler des requêtes de lecture pour répondre avec précision aux questions de vos utilisateurs.

---

## **🛠️ Comment connecter**

1. Accédez à vos **Paramètres de discussion** \> **Intégrations** \> **Base de données SQL**.
2. Cliquez sur **Connecter** et renseignez les informations de votre base de données :
   - **Nom et description :** un nom convivial et une description pour aider l'IA à comprendre les données contenues.
   - **Devise (facultatif) :** la devise principale utilisée pour les valeurs monétaires dans vos tables, ce qui aide l'IA à formater et interpréter les données financières.
   - **Hôte et port :** l'adresse et le port de votre serveur de base de données (par défaut `5432` pour PostgreSQL).
   - **Base de données :** le nom de votre base de données.
   - **Utilisateur et mot de passe :** vos identifiants de base de données (idéalement en lecture seule).
   - **Mode SSL :** sélectionnez `require` ou `disable`.
3. Cliquez sur **Tester** pour vérifier les identifiants et voir combien de tables sont trouvées.
4. Cliquez sur **Enregistrer** pour terminer.

---

## Remplir les informations de connexion

Cliquez sur le bouton Connecter pour ouvrir le formulaire et renseigner les champs suivants :

| Champ | Exemple de description |
| --- | --- |
| Nom | Un nom convivial pour identifier cette connexion |
| Description | Une description de ce que fait la base de données (aide Attlas à comprendre le contexte des données) |
| Devise | La devise principale utilisée pour les données monétaires dans cette base de données (par exemple, USD, EUR), facultatif |
| Hôte | L'adresse de votre serveur de base de données. [db.example.com](http://db.example.com) ou 123.45.67.89 |
| Port | Le port de connexion (5432 par défaut pour PostgreSQL) |
| Base de données | Le nom de votre base de données |
| Utilisateur | Le nom d'utilisateur de connexion |
| Mot de passe | Le mot de passe associé à l'utilisateur |
| Mode SSL | Le niveau de sécurité de la connexion SSL |

## **⚙️ Gérer votre intégration**

Cliquez sur **Paramètres** sur la carte de votre base de données SQL active pour gérer les éléments suivants :

- **Sélectionner les tables à partager :** vous n'avez pas à tout partager. Activez/désactivez les tables ou vues spécifiques auxquelles l'IA est autorisée à accéder.
- **Actualiser le schéma :** si vous ajoutez ou modifiez des colonnes/tables dans votre base de données, cliquez sur **Actualiser** pour synchroniser les modifications avec l'IA.
- **Désactiver temporairement :** activez ou désactivez l'intégration à tout moment à l'aide du bouton à bascule.
- **Déconnecter :** cliquez sur **Déconnecter** pour supprimer complètement la connexion à la base de données.

## **💡 Bonnes pratiques pour le schéma de base de données**

Pour aider Attlas à écrire des requêtes précises et à effectuer des calculs exacts, tenez compte de ces conseils de conception de schéma :

- **Utilisez des types de colonnes précis :** si vous souhaitez que le chat effectue des calculs, tels que des sommes, des moyennes ou des ratios, sur des données numériques, assurez-vous que ces colonnes utilisent un type numérique, tel que `numeric`, `decimal`, `integer` ou `float`. Évitez de stocker des nombres sous forme de texte, comme `varchar` ou `text`, car cela rend les calculs plus difficiles et moins précis.
- **Ajoutez des commentaires de colonne :** utilisez les commentaires de base de données sur vos tables et colonnes. Ajouter une brève description directement à une colonne, par exemple `COMMENT ON COLUMN users.status IS 'Active statuses: active, suspended, pending'`, fournit un contexte essentiel qui guide Attlas dans la rédaction des requêtes SQL correctes.

---

## **🔒 Bonnes pratiques de sécurité**

Pour protéger vos données, nous vous recommandons vivement :

- **Utilisez un utilisateur en lecture seule :** créez un utilisateur de base de données avec uniquement les autorisations `SELECT`. Attlas n'a pas besoin d'accès en écriture (`INSERT`, `UPDATE`, `DELETE`).
- **Gardez SSL activé :** nous vous recommandons de configurer `sslmode` sur `require` pour chiffrer les données en transit.
