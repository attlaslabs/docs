A integração de banco de dados SQL (compatível com PostgreSQL) permite conectar um banco de dados à sua sala de chat. Uma vez conectado, o Attlas poderá ler o esquema das tabelas autorizadas e formular consultas de leitura para responder com precisão às perguntas dos seus usuários.

---

## **🛠️ Como conectar**

1. Vá para **Configurações do Chat** \> **Integrações** \> **Banco de dados SQL**.
2. Clique em **Conectar** e preencha os detalhes do seu banco de dados:
   - **Nome e Descrição:** Um nome amigável e uma descrição para ajudar a IA a entender quais dados estão armazenados.
   - **Moeda (Opcional):** A moeda principal usada para valores monetários nas suas tabelas, ajudando a IA a formatar e interpretar dados financeiros.
   - **Host e Porta:** O endereço e a porta do servidor do banco de dados (padrão é `5432` para PostgreSQL).
   - **Banco de dados:** O nome do seu banco de dados.
   - **Usuário e Senha:** Credenciais do banco de dados (de preferência, apenas leitura).
   - **Modo SSL:** Selecione `require` ou `disable`.
3. Clique em **Testar** para verificar as credenciais e ver quantas tabelas foram encontradas.
4. Clique em **Salvar** para concluir.

## Preencha os detalhes da conexão

Clique no botão Conectar para abrir o formulário e preencher os seguintes campos:

| Campo | Descrição de exemplo |
| --- | --- |
| Nome | Um nome amigável para identificar esta conexão |
| Descrição | Uma descrição do que o banco de dados faz (ajuda o Attlas a entender o contexto dos dados) |
| Moeda | A moeda principal usada para dados monetários neste banco de dados (ex.: USD, EUR), opcional |
| Host | O endereço do seu servidor de banco de dados. [db.example.com](http://db.example.com) ou 123.45.67.89 |
| Porta | A porta de conexão (5432 por padrão para PostgreSQL) |
| Banco de dados | O nome do seu banco de dados |
| Usuário | O nome de usuário para conexão |
| Senha | A senha associada ao usuário |
| Modo SSL | O nível de segurança da conexão SSL |

## **⚙️ Gerenciando sua integração**

Clique em **Configurações** no cartão do seu banco de dados SQL ativo para gerenciar o seguinte:

- **Selecionar tabelas para compartilhar:** Você não precisa compartilhar tudo. Ative ou desative quais tabelas ou visualizações específicas a IA tem permissão para acessar.
- **Atualizar esquema:** Se você adicionar ou editar colunas/tabelas no seu banco de dados, clique em **Atualizar** para sincronizar as alterações com a IA.
- **Desabilitar temporariamente:** Ative ou desative a integração a qualquer momento usando o botão de alternância.
- **Desconectar:** Clique em **Desconectar** para remover completamente a conexão do banco de dados.

## **💡 Melhores práticas para esquema de banco de dados**

Para ajudar o Attlas a escrever consultas precisas e realizar cálculos exatos, considere estas dicas de design de esquema:

- **Use tipos precisos de colunas:** Se você deseja que o chat realize cálculos, como somas, médias ou proporções, em dados numéricos, certifique-se de que essas colunas usem um tipo numérico, como `numeric`, `decimal`, `integer` ou `float`. Evite armazenar números como texto, como `varchar` ou `text`, pois isso dificulta e torna os cálculos menos precisos.
- **Adicione comentários nas colunas:** Utilize comentários de banco de dados em suas tabelas e colunas. Adicionar uma breve descrição diretamente em uma coluna, por exemplo `COMMENT ON COLUMN users.status IS 'Active statuses: active, suspended, pending'`, fornece um contexto crucial que orienta o Attlas a escrever as consultas SQL corretas.

---

## **🔒 Melhores práticas de segurança**

Para manter seus dados seguros, recomendamos fortemente:

- **Use um usuário somente leitura:** Crie um usuário de banco de dados com apenas permissões de `SELECT`. O Attlas não precisa de acesso de gravação (`INSERT`, `UPDATE`, `DELETE`).
- **Mantenha o SSL ativado:** Recomendamos definir `sslmode` como `require` para criptografar os dados em trânsito.
